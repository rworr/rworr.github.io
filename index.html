Operating Systems

Undergraduate Research

I am currently working with Prof. Wojciech Golab to identify and implement potential areas of improvement for in-memory B-Trees within multi-core systems.

SAP: Software Developer, Kernel Engine Team

During my work term at SAP, I researched, designed, and implemented a partially lock-free dynamic memory allocator in C++. I began by studying various research papers and current company code, and designed the memory allocator by incorporating ideas from the papers and current code, and brainstorming data structure design with my co-workers. I also created benchmarks to compare the current memory manager, prototype, and malloc on Windows, and wrote Python code to simulate memory movement within designs using data obtained by tracing allocations in code. Ultimately, the final implementation showed a 10% performance increase over malloc on 64-bit systems and a 20-30% performance increase over malloc on 32-bit systems.

Lock-Free Data Structures

I am continuing to explore the area of lock-free technologies on my own time. I have implemented some basic lock-free data structures such as stacks and linked lists in C++, and have conducted performance analysis against traditional locking methods using Intel's VTune Amplifier. [Github Repo]

Virtual Machine in C#

I have completed a very rudimentary virtual machine using C#, and a custom assembly language and assembler to accompany it. I followed a methodology similar to this codeproject article, by implementing the same functionality but developing the code on my own. The machine is a 16-bit machine, using 5 registers, and 64K of addressable memory, 4K of which is used for a basic console capable of displaying characters and colour attributes. [Github Repo]

Compilers and Languages

OptumSoft: Compilers/Languages SW Eng.

At OptumSoft, I worked on refactoring the internal structure of their compiler to provide earlier expansion of constructs in the abstract syntax tree, with the goal of providing a simplified execution path and ensuring clarity of which case is being handled. Additionally, I gained experience working in a linux environment, including debugging using GDB.

Generic Fuzzer

As an external side project to the Compilers course I took at UWaterloo, I worked on developing a generic fuzzer using Python. The goal of the fuzzer was to accept a language specification file, and output randomized but valid files. The fuzzer worked well for most lanugage specifications, but there were some edge cases I didn't get a chance to look at. [Github Repo]

Taught by Alex Aiken of Stanford University via Coursera.org, this course outlined the aspects of a compiler for a simple object-oriented language called COOL, including lexical analysis, parsing, semantic analysis and type checking, code generation, optimization, and garbage collection.

Coursera: Pattern-Oriented Software Architectures

Taught by Douglas Schmidt of Vanderbilt University via Coursera.org, this course looked at patterns, which are general solutions to common problems in software design, and focused on the patterns used in concurrent and networked software. In addition to an overview of concurrency, networking, and the associated patterns and frameworks, the course also featured a look at the Android and ACE frameworks, and a look inside JAWS, an open-source web server.

Coursera: Heterogeneous Parallel Programming

Taught by Wen-mei W. Hwu of the University of Illinois at Urbana-Champaign via Coursera.org, this course covered the parallel execution model, memory models for locality, parallel algorithm patterns, overlapping computation and communication, and scalable programming using CUDA in computing clusters. The course included 5 parallel programming assignments written in CUDA C (vector addition, simple and tiled matrix multiplication, reduction and scan algorithms).

Coursera: Interactive Programming in Python

Taught by Joe Warren, Scott Rixner, John Greiner, and Sephen Wong of Rice University via Coursera.org, this course covered an introduction to python with a focus on simple game development. Topics included object-oriented programming in Python, in-game physics, and basic animations. Assignments were conducted in the codeskulptor environment, and included a variety of simple games, such as pong, blackjack, and RiceRocks (Asteroids). On codeskulptor, press Alt-R to play RiceRocks. Use the arrow keys to control movement, and the space bar to fire missiles.

Udacity: Computer Science 101

Taught by David Evans of the University of Virginia via Udacity.com this course gave an introduction to Python. The course project involved construction of a web crawler and how to extract and follow links in HTML pages, and basic search engine functionality using the PageRank algorithm to prioritize crawled pages.

Coursera: Cryptography I

Taught by Dan Boneh of Stanford University via Coursera.org, this course covered a variety of cryptographic principles and primitives including block ciphers (3DES, AES), message integrity (MAC), and public key encryption (Diffie-Hellman, RSA). The course also introduced mathematical definitions of concepts such as semantic security and perfect security.

Coursera: Image and Video Processing

Taught by Guillermo Sapiro of Duke University via Coursera.org, this course covered image and video compression, spatial processing, image restoration, image segmentation, and image and video inpainting and special effects. Applications of image and video processing, such as medical imaging and image transfer, were also discussed.
